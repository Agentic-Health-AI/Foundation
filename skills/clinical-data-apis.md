# Clinical Data APIs

FHIR servers, OMOP tools, and clinical data interoperability APIs.

- **[HAPI FHIR](https://hapifhir.io/)**: Complete open-source FHIR implementation with a deployable server, client library, and validation tools. The most widely used FHIR framework for building standards-compliant healthcare applications.

- **[ClinicalTrials.gov API v2](https://clinicaltrials.gov/data-api/api)**: REST API providing access to global clinical trials data, including study protocols, eligibility criteria, and recruitment status. Free to use for research and application development.

- **[OHDSI / OMOP CDM](https://github.com/OHDSI)**: Open-source standards and tools for observational health data, enabling consistent analysis across diverse healthcare datasets. Includes detailed documentation, R packages, and best practices for implementing the Common Data Model.

- **[Medplum](https://github.com/medplum/medplum)**: Open-source FHIR developer platform offering a hosted FHIR server, SDKs, and UI components. Simplifies building compliant healthcare applications with built-in APIs and data management tools.

- **[Metriport](https://github.com/metriport/metriport)**: Universal healthcare data API that aggregates and normalizes clinical data across multiple EHR systems. Provides FHIR-compliant access to patient records with unified data models.

- **[HL7 FHIR Public Test Servers](https://confluence.hl7.org/display/FHIR/Public+Test+Servers)**: Free public FHIR test environments maintained by the community for development and testing. Includes multiple server implementations to validate FHIR implementations across different platforms.
